# Accessibility

## Commitment

7ice experiences must be usable with keyboard, screen readers, zoom, touch, low vision, cognitive differences, and varying network conditions. The target is WCAG 2.2 AA unless a stricter contractual or legal requirement applies. Accessibility is a shared design, content, engineering, and QA responsibility.

## Non-negotiable requirements

- Use semantic HTML and a logical heading, landmark, reading, and focus order.
- Make every function keyboard-operable with visible focus and no keyboard trap.
- Provide accessible names, instructions, error summaries, status announcements, and correctly managed dialog focus.
- Meet contrast and target-size requirements; do not encode meaning by color, motion, position, or sound alone.
- Support responsive reflow, 200% zoom, user text scaling, reduced motion, and sufficient time controls.
- Supply accurate alternatives for informative media and avoid inaccessible document-only critical content.

## Assurance

Use design review, component-level checks, automated tests, keyboard testing, assistive-technology spot checks, and user research. Record exceptions with impact, workaround, owner, and target resolution—not as permanent acceptance.
